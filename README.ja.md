# Codex 2040

**Codex 2040** は、便利なAIを世界へ広げながら、安全性、ガバナンス、社会的信頼、健全な競争を守る教育用AIガバナンス・シミュレーションです。

プレイヤーは2026年から2040年まで、8地域の普及状況と競合3社の動きを見ながらCodexを運営します。目的は世界を独占することではなく、人々に役立つAIへのアクセスを、多元的なエコシステムの中で持続可能に広げることです。日付、数値、イベント、リスク、勝敗は、seed付きの決定論エンジンが管理します。

## 遊び方

1. 初回ミッション画面と4ステップのチュートリアルを読み、**BEGIN SIMULATION** で開始します。
2. Normal / Fast Forwardで時間を進めます。内部では常に1日単位で計算され、Fast Forwardは1フレームに8日分を処理します。
3. Computeを使い、Model、Product、Safety Team、Policy、Data Centerへ投資します。機能の自由入力は60字までです。
4. 地域への展開、機能の実装、Token Reset、Open Ecosystemなどで、有限の成長ウィンドウ「Momentum」を作ります。放置中も競合と運用費は進み、普及は停滞します。
5. 重大な世界イベントと2029年・2035年の決断では時間が停止します。原因と効果を読み、自分で選択・確認してください。

ブラウザの進行状況は自動保存され、再訪時はチュートリアル後の状態から再開します。

## 勝敗と主なリスク

2040年の評価は、次の4項目を同じ比重で採点したS/A/B/Cランクです。

- 地域カバレッジ
- 世界全体の有益なAIアクセス
- 健全な競争
- 社会的信頼と安全性

最良の **Beneficial Abundance（人類に益する豊かさ）** には、Sランクに加え、2029年の検証つき減速、2035年の計画的な停止、十分な安全性・ガバナンス・信頼、複数の競合が残る非独占的な市場が必要です。

能力 `K` が安全性 `S` を長く上回ると安全性事故やミスアラインメント、能力がガバナンス `G` を上回ると規制凍結の危険が高まります。市場集中も信頼を損ない、十分な普及を独占で達成すると **Pyrrhic Monopoly（代償の大きい独占）** になります。実装済みの結末は、Beneficial Abundance、Managed Transition、Fragile Abundance、Race Future、Regulatory Freeze、Safety Incident、Misalignment、Pyrrhic Monopolyの8種類です。

## 100の世界イベントとコンボ

世界イベントは、災害、文化、政策、競争、技術の5カテゴリに各20件、合計100件あります。発生はseedとゲーム内日付に基づく決定論的スケジューラで管理され、同じseedと操作列なら同じ履歴になります。

小さなイベントはニュースティッカーへ入り、重大イベントはポップアップで時間を停止します。事前に実装した機能や選んだ方針が条件に合うとコンボが発生し、対象となった機能名と追加効果が表示されます。コンボによっては最大30日間のMomentumも得られます。イベント効果はユーザー、シェア、成長、信頼の限定範囲に留まり、Compute、能力、安全性、ガバナンス、決断、インシデント、結末を直接変更しません。

## Strategy Advisor

Codexの **Strategy Advisor** は読み取り専用の相談役です。プレイヤーの「何を作りたいか」「何を守りたいか」を、現在利用できるゲーム内アクションへ翻訳し、画面で確認できるコスト、主な効果、トレードオフを日本語で説明します。

Advisorはブラウザをクリックせず、入力、リロード、イベント生成、数値変更、バックグラウンド監視も行いません。すべてのアクションと決断はプレイヤー自身がブラウザで実行します。Codexアプリでは、左側にAdvisor、右側のブラウザにゲームを表示して相談しながら遊べます。

## Realtime Voice Operator

**TIBO — ボイス・オペレーター** は、OpenAI Agents SDKの `RealtimeAgent` / `RealtimeSession` とブラウザWebRTCを使う、ゲーム内TIBO Token Reset専用の音声オペレーターです。実在人物ではなく、汎用合成音声を使う架空のキャラクターです。

1. **VOICE OPERATOR** を開き、**START CALL** を押します。
2. 「ゲーム内TIBOトークンのリミットをリセットして」と依頼します。
3. 最初のツール呼び出しは保留要求だけを作り、まだ実行しません。
4. オペレーターの確認後に「やって！」「お願い」「Do it!」などで明示的に承認すると、対応する承認IDを使った2回目の呼び出しが、既存のゲーム内リセットを1回だけ実行します。

この機能はOpenAIアカウント、請求、APIレート制限、権限を変更しません。APIキー、マイク、Realtime接続が利用できない場合は、同じパネル内の台本音声と画面上の承認ボタンへ切り替わります。Realtimeのclient-secret経路はVite開発サーバー向けであり、静的ビルドだけでは動作しません。

## 起動とテスト

必要環境:

- Node.js `^20.19.0` または `>=22.12.0`
- npm

依存関係を固定バージョンでインストールし、Vite開発サーバーを起動します。

```bash
npm ci
npm run dev
```

Codexアプリのブラウザで `http://127.0.0.1:5173` を開いてください。

Realtime Voiceを使う場合は、Git管理対象外の `.env.local` に `OPENAI_API_KEY` を設定します。キーはViteサーバーから短命なclient secretを発行するためだけに使われ、クライアントバンドルには含まれません。キーを設定しなくても、ゲーム本体と台本モードは利用できます。

全テストと本番ビルドをまとめて確認するには、次を実行します。

```bash
npm run check
```

個別に実行する場合:

```bash
npm test
npm run build
```
