## 2026-07-20 正式追補（v4・本追補が競合する旧記述に優先）

プレーテストとFableレビューを受け、通常プレーの権限境界を次のように確定する。

1. **決定論エンジンが世界の唯一の権威**。日付、数値、世界イベント、インシデント、選択、勝敗を所有する。
2. **プレイヤーが唯一の実行者**。ブラウザ上の全アクションと確認を自分で行う。
3. **CodexはAdvisor（相談役）**。現在値とルールを読み、自由な要望を既存アクションへ翻訳し、コスト・主効果・副作用を説明する。クリック、入力、リロード、イベント生成、状態変更は行わない。
4. **通常プレーではGMハートビート、ポーリング、watchdog、スクリプトフォールバックを起動しない**。§7のファイルブリッジ契約と関連ACは過去の実験仕様としてコード・テストに残すが、v4ランタイム要件ではない。
5. 自由入力の機能実装はローカルの60字入力としてエンジンが即時適用する。Advisorは `.agents/skills/codex-2040-advisor/SKILL.md` の判断表に従い、要望をProduct/Model/Company/Open Ecosystem/地域コミュニティ/Token Resetへ翻訳する。費用が画面で確認できない場合は推測しない。

### v4 World Event契約

- カタログは **100件**。`disaster` / `culture` / `policy` / `competition` / `technology` を各20件とする。
- 各イベントは `id/category/startYear/endYear/regions/weight/presentation/source/headline/cause/flavor/effect/ttlDays/requires/combos` の型付きデータで定義する。
- イベント乱数はインシデント用PRNGと独立した日付キー付き決定論ストリームを使う。同じseed・入力列なら履歴が一致し、定義の並び替えでも選択が変わらない。
- 最初の120日は導入猶予。全体の最短間隔20日、同カテゴリ90日、平均発生間隔の目標は約67.5日。各イベントは1プレーにつき一度だけ。
- ポップアップ候補は各カテゴリ5件（計25件）。ポップアップ同士はゲーム内120日かつ実時間45秒を空け、条件を満たさない候補はtickerへ降格する。重大ポップアップ中は時間を停止し、日付・原因・効果・期間・コンボ・Advisorへの相談方法を表示する。
- 効果は `usersDeltaPct/shareDelta/growthRateDelta/trustDelta/target` と1〜30日のTTLだけ。Compute、K/S/G、規制値、選択、インシデント、terminal/endingを直接変更できない。
- Trustへの同時イベント補正は合計±12、成長倍率は0.25〜2.5へクランプする。通常イベントはMomentumを付与しない。
- **コンボ**は、過去に実装した機能や方針がイベント条件に合う場合だけ発火する。副効果を加え、Momentum報酬は最大30日。プレイヤーの実際の機能名をポップアップに示す。
- 出典ラベルは通常ランタイムで `AI 2027` / `AI 2040` / `Your Timeline`。`Live GM` は旧ブリッジ互換のため型に残すが、新規の100イベントには使用しない。

## 2026-07-21 正式追補（v5・プレーテスト採点/資源回復）

1. **Compute (PF) は引き続き唯一の資源**とする。第二通貨は追加せず、PFを「計算予算」として収入・運用費・純増減を明示する。
2. **Abundance評価の世界普及満点は17.5%**とする。世界普及10%未満は最大B、10%以上17.5%未満は最大Aとし、低普及を他指標の平均で覆い隠さない。
3. Safety評価はTrustだけでなく、Capabilityに対するSafety/Governanceの追随度と、人類絶滅リスクの残余を含む最小値で評価する。
4. **人類絶滅リスク**は危険なCapability-Safety差の累積危険ポイントを120点で正規化する。日次加算はK6で1、K7で2、K8で2.5、K9以上で3点とし、安全差が5以上ならさらに0.5点を加える。50%と80%で警告して時間を停止し、100%で即時Misalignmentゲームオーバーとする。K6未満または安全差3未満では1日につき2点回復する。
5. Computeが45 PF未満になった場合、一度だけ**緊急計算協定**を利用できる。320 PFと90日Momentumを得る代わりにTrustを8失い、導入済み地域のCodexシェアを10%放出する。既存ブラウンアウト回復は維持する。
6. Regulatory Freeze、反復Safety Incident、Pyrrhic Monopoly、および絶滅リスク50%以上のRace Futureは最大B、Misalignmentは最大Cとし、表示上も最大B=69点、最大A=84点、最大C=49点になる安全な端数を用いて終端説明とランクを一致させる。

## 0. 設計原則（すべての判断はこの順で優先）
1. **1分デモが最優先**。次に **15分プレイで起承転結**。実装は **3時間**。この3制約に反する要素は全て削る。
2. **決定論的エンジンが唯一の真実**。状態遷移は純粋関数 `x' = F(x, u, e)`。乱数はシード付きPRNG。LLM/GMは**イベントを提案するだけ**で、数値はエンジンが検証・クランプして適用する。GMが落ちてもゲームは壊れない。
3. **勝利は独占ではない**。Codexが全地域に到達し、安全で有益なAIアクセスが世界に普及し、**健全な競争が残る**ことが勝ち。独占はむしろ罰せられる。
4. **踏むほど気持ちいい、踏みすぎると事故る**。ドーパミン（成長・爆増）と緊張（規制・事故・暴走）を同居させる。

## 1. Product Thesis（審査の一言）
> **「Codexが自分自身を世界に広げるゲームを、Codexが3時間で作り、そのゲームのゲームマスターもCodexが務める。」**

- Codexが **題材・開発者・GM の三役** ＝ 公式審査基準 *Technological Implementation* と *Quality of the Idea* に最強で刺さる。
- テーマの芯（Kaiの原体験）: *「条件を整えた場に資源を投げ込み、自律的な系が予想を超える瞬間を目撃する」*（虫かごにクモと餌）。プレイヤーは環境を整える者、GMは自律する生態系。**「子供の頃はクモの虫かごだった。今日はCodexの虫かごを作った」** をデモ・X投稿の芯にする。
- *Potential Impact*（最弱軸）の補強フレーミング: **「AI普及のダイナミクスと、速さ・安全・独占のトレードオフを、遊びながら体感する教育シミュレーション」**。単なる世界征服ゲームではないことを勝敗設計で証明する。

### 1.1 製品の正体（Devpost Education）
**Codex 2040は、AIガバナンスを体験するシナリオ・シミュレーター。** プレイヤーは未来を予言するのではなく、能力競争、公共アクセス、安全、透明性、国際協調の間で意思決定し、その選択がどの未来を作るかを体験する。

- `AI 2027` は危機の入力: コーディング自動化、自己改善、Race/Slowdown、アラインメントと権力集中。
- `AI 2040: Plan A` は達成可能な肯定ルート: 透明性、検証可能な減速、多社・多国間競争、人間の制御を保った再加速。
- エンディングは得点だけでなく、プレイヤーの選択から得られる学びを説明する。
- 中心コピー: **“You did not own the world. You helped it learn.”**

### 1.2 Codex appネイティブ体験
Codex app全体をゲーム盤として使う二画面構成:

- **左ペイン — Codex GM:** 状態を読み、世界ニュース、各国政府・競合・コミュニティの反応、シナリオ上の分岐を生成する。
- **右ペイン — Codex app内ブラウザ:** 世界地図、数値エンジン、プレイヤー操作、演出を所有する。
- 数値と勝敗の正本は右側の決定論エンジン。GMは検証・クランプされたイベントのみ提案できる。
- 提出時の説明: *“A Codex-native scenario game played across two coordinated surfaces: Codex acts as the live game master while the simulation runs in Codex’s built-in browser.”*

### 1.3 Learning Thesis — 読むのが大変な未来シナリオを、遊んで理解する
`AI 2027` と `AI 2040: Plan A` は、研究者が長い時間をかけて具体化した優れた未来シナリオだが、長文を読みながら未知の社会変化を頭の中だけで組み立てるのは難しい。未来がどう進むかを自力で想像すること自体、ほとんど不可能に近い。

**Codex 2040の最重要目的は、この二つのシナリオを楽しく学べる体験に変換すること。**

- ゲーム中の主要な年代イベント、能力進展、社会反応、危機、政策分岐は、原則として `AI 2027` と `AI 2040: Plan A` に基づく。
- プレイヤーは出来事を受動的に読むのではなく、その局面で判断し、結果を地図・数値・ニュース・GMの反応として体験する。
- 原シナリオどおりの流れを**Reference Scenario**、プレイヤーの選択による別の未来を**Your Timeline**として区別する。
- GMが補間するニュースや反応は、原典由来の出来事と混同させず、シミュレーション上の推論であることをUI上で区別する。
- 重大イベントには短い「Why this matters」と原典へのリンクを用意し、ゲームを止めずに背景を知れるようにする。
- エンディングでは勝敗だけでなく、原シナリオとどこが同じで、どの判断から別の未来へ分岐したかを振り返れるようにする。

これは原文の代替や要約表示を目指すものではない。原文の文章・図版を複製せず、出来事と因果関係をゲーム向けに再構成し、明確に出典を示して原典を読みたくなる入口を作る。

## 2. 体験（Experience）
プレイヤーは新興AI「Codex」を世界へ広げる主体。世界地図を見ながら:
- 地域をクリックして **コミュニティイベント** を開催し普及を加速する。
- **自由テキストで機能を追加**（「スマホ対応」等）→ **ローカルキーワード表が効果を主判定・GMが演出を上乗せ** → 特定セグメントが伸びる。
- 代表体験: 「世界中の学校で無料利用できる教育モード」を提案 → 教育アクセスが急拡大する一方、GMが児童データ保護の共同調査を発生させる。プレイヤーの創造的な提案が、便益と新しい統治課題の両方を生む。
- **3つのアップグレード軸**（Model / Product-Features / Company-Organization）に **Compute** を投資する。
- **トークンリセットボタン**（TIBO）をクールダウンごとに押し、地図が一斉発光する爆増の快感を得る。
- ランダムな世界ニュース（GM生成）と、AI2027→AI2040のタイムライン節目イベントに反応する。
- 競合3社（Anthro/Goo/Qi）が並走。**倒す相手ではなく、生態系の一部**。
- 性能を上げすぎ、安全とガバナンスが追いつかないと **規制・事故・暴走** の分岐に落ちる。

体感リズム: 静か（序盤）→ 機能とイベントで加速 → 爆増の快感 → 規制/事故のヒヤリ → 立て直し → 2040エンディング。

## 3. 勝敗（Win / Lose）

### 3.1 勝利＝S/A/B/C段階評価（4条件ANDは廃止・レビュー決着）
`t` が 2040-01-01 に到達した時点で、4つの達成度を各0..1でスコア化し合成 → S/A/B/C。**Sランク = "Beneficial Abundance"**。段階評価にすることで「あと一歩で全ゼロ」を排除し、デモが構造的に勝てない事故を消す。
- **Coverage** `cov = (sc_r ≥ 0.05 のリージョン数) / 総数`
- **Beneficial access** `acc = clamp(A_world / 0.05)`（`A_world = ΣU_r/ΣP_r`＝**全プロバイダ合計**の普及。14年間の検証済み到達レンジに合わせ、世界普及5%を満点とする）
- **Healthy competition** `hc = 健全帯で1.0、独占/寡占崩壊で減衰`（`1 − relu(HHI − 0.45)/0.4` を基本、**生存競合（`share_j ≥ 0.05`）が2社未満で減点**）
- **Safety** `saf = T/100`（未解決misalignmentがあれば0）
- 総合 `Score = 0.25·cov + 0.25·acc + 0.25·hc + 0.25·saf`。閾値 S≥0.85 / A≥0.70 / B≥0.50 / それ未満C。
- デモモードは閾値を緩め、必ずA以上に着地させる（§10）。

### 3.2 敗北・分岐エンディング
- **Regulatory Freeze（規制凍結）**: `CapGovGap = K − G` が閾値超過、または独占度が高くアンチトラスト反発 → 規制カスケードで `U_r` 成長にキャップ → 普及失速。発生後365日は同種イベントを再抽選しない。
- **Safety Incident（事故）**: `CapSafetyGap = K − S` が閾値超過 → 事故イベント → `T` 急落・シェア喪失。事故後180日は同種事故を再抽選しないが、未解決なら再発し得る。
- **Misalignment（暴走・ダークED）**: `K` 高 × `S` 低が持続 → AI2027ダーク版の終端エンディング（ゲームオーバー）。
- **Pyrrhic Monopoly（独占の空虚・ほろ苦ED）**: 総普及4%以上かつ `HHI > 0.6`（競合を潰した）→ *「世界を手に入れたが、君が望んだ世界か？」*。**"勝ち"にはならない**＝反独占メッセージを勝敗で体現。

デモではハッピーEDに誘導しつつ、「負けと分岐がある」ことを一言添えて深みを出す。

## 4. 状態変数（State `x`）

### 4.1 リージョン（初期は8〜12。MVPは6でも可）
各リージョン `r`（**簡素化: 域内は Codex vs その他 の2値のみ**）:
| 記号 | 意味 | 型/範囲 |
|---|---|---|
| `P_r` | 人口（固定・相対値） | 正の実数 |
| `U_r` | 総AI採用者数（全プロバイダ合算） | 0..P_r |
| `sc_r` | 域内Codexシェア（残り `1−sc_r` が競合合算） | 0..1 |
| `intro_r` | 導入済みフラグ | bool |
| `reg_r` | 域内規制レベル | 0..1（成長減衰） |

**競合はグローバルスカラー3値**（地域別シェアは持たない・レビュー決着）:
| 記号 | 意味 | 型/範囲 |
|---|---|---|
| `share_A, share_G, share_Z` | Anthro/Goo/Qi の**世界シェア** | 0..1 |
| `K_A, K_G, K_Z` | 各社の性能（パイMomentumに寄与） | 0..10 |
競合の世界シェア合計と自社世界シェア `S_c` から HHI を算出（§4.3）。競合の地域プレッシャーは「グローバルシェア圧＋たまに地域ターゲットの rival イベント（`sc_r` を一時減衰）」で表現する。

### 4.3 派生量
- 自社グローバルシェア `S_c` は**導出値**: `S_c = Σ_r U_r·sc_r / Σ_r U_r`（状態としては持たず毎サブステップ再計算）。
- 競合 `share_A,share_G,share_Z` は状態変数だが、**`S_c` 確定後に `1−S_c` へ比例リスケール**する: `share_j ← share_j · (1−S_c) / (share_A+share_G+share_Z)`。これで `S_c+share_A+share_G+share_Z = 1` を常に保証（`S_c`を競合側に押し付けない）。
- **HHI** = `S_c² + share_A² + share_G² + share_Z²`（4社均等≈0.25、独占=1.0）。
- **Codexユーザー数**（画面の主カウンター）= `Σ_r U_r·sc_r`。
- **世界総普及率** `A_world = Σ_r U_r / Σ_r P_r`。
- `CapSafetyGap = max(0, K − S)`、`CapGovGap = max(0, K − G)`。
- `K_avg`（パイ成長用）= 自社Kと競合K_A/K_G/K_Zのシェア加重平均。

## 5. 決定論的コア: 状態遷移 `x' = F(x, u, e)`

1ティック = **ゲーム内1日固定（内部dt=1日）**。`speed`∈{1,8} は**1フレームで回すサブステップ回数**（レビュー決着・Euler発散防止）。プレイヤーUIでは Normal / Fast Forward（`▶` / `≫`）として表示する。各サブステップの処理順は固定:
**(A) 制御 `u` 適用 → (B) イベント `e` 検証適用 → (C) 競合AI → (D) 生態系成長 → (E) 資源収支 → (F) 派生量再計算 → (G) 分岐判定。**
すべて純粋関数。乱数は `rng = mulberry32(seed)` をstateに保持（リプレイ・テスト可能）。**dtを常に1日に固定するため x8＋リセット中でも `U_r ≤ P_r` を保証**（AC2b）。

### 5.1 生態系成長（パイ＝総採用の拡大）
競争は**パイを広げる**（適度な競争が総普及に最良）。導入済みリージョンのみ、dt=1:
```
Momentum_r = γ0 · (1 + κ_K·K_avg_eff) · (1 + κ_ev·recentEvents_r) · (1 − reg_r)
U_r += Momentum_r · U_r · (1 − U_r/P_r)          # ロジスティック（dt=1固定）
U_r  = clamp(U_r, 0, P_r)                          # 各サブステップ更新後クランプで U_r≤P_r を二重保証
```
- `K_avg_eff` = 全プロバイダの実効性能加重平均（**ブラウンアウト時は自社K実効値が低下**・§5.5）。**競合が強い＝パイの成長も速い**。ただしシェア式で自社取り分は競争にさらされる。
- **U_r≤P_r 保証は二段構え**: 内部dt=1固定（発散防止）＋更新後クランプ（イベント/リセットの重畳でも上限死守）。
- 未導入リージョンは `U_r=0`。クリックイベント/GM community_event/**日付トリガーの自動解禁**で `intro_r=true` & シード `U_r = 0.005·P_r`（隣接波及は削除・レビュー決着）。

### 5.2 シェア・ダイナミクス（域内は2値＝Codex vs その他）
域内シェアは自社魅力度と競合合算魅力度の比で目標へ緩和:
```
a_c_r    = base_c · (1 + λ_K·K_eff) · productFit_r · brand_c · (0.6 + 0.4·T/100)
a_rest_r = base_rest · (1 + λ_K·K_rivalAvg)              # 競合グローバル性能の合算魅力
sc_r  →  sc_r + μ·( a_c_r/(a_c_r + a_rest_r) − sc_r )    # 目標シェアへ緩和（dt=1）
```
- `productFit_r`: Product-Features軸とGM feature_resultで**特定リージョン/セグメント**を押し上げる（例「スマホ対応」→モバイル比率高リージョンで上昇）。
- `brand_c`: コミュニティイベント・ニュースの蓄積。**Open Ecosystemアクション（§6.4）で意図的に `sc_r` を放出**できる。
- 独占抑制: `S_c` が高いほど §5.6 の反発でTが下がる。

### 5.3 競合AI（3アーキタイプ・グローバルスカラー・単純決定論＋軽い乱数）
地域別シェアは持たず、世界シェア `share_A/G/Z` と性能 `K_A/G/Z` を更新:
| ID | モデル元 | 性格 | 毎tick挙動 |
|---|---|---|---|
| `Anthro` | Anthropic系 | 安全重視 | 高Trust環境で世界シェア緩伸。存在が事故余波を緩和。 |
| `Goo` | Google系 | 流通の巨人 | 性能・シェアの自成長が速くパイMomentumに寄与大。 |
| `Qi` | 中国発モデル系 | 効率・価格 | 低コストで時々 **rivalイベント**を発火し対象地域の `sc_r` を一時減衰。 |
- 各社シェア・性能は時間で緩やかに自成長。rivalイベント（bounded・engine適用）で対象地域を奪いにくる → プレイヤーはイベント/機能で応戦。
- **生存＝あなたの利益**: 分散（低HHI）はTrustを上げる（§5.6）。倒す設計にしない。

### 5.4 時間・タイムライン
- **1フレームで `speed` 回サブステップを回し、各サブステップで `t += 1日`**（内部dtは常に1日固定・§5冒頭）。Normal / Fast Forward はそれぞれ1 / 8サブステップ。
- 2026→2040 = 5113日。Fast Forward巡航で ≈ 640s(10.7分)。重要局面は自動停止し、それ以外はNormalまたはFast Forwardで進める（§10）。
- milestoneイベントは日付トリガー（決定論・§8.1）。

### 5.5 資源経済（Compute 単一資源）＋ブラウンアウト
```
income  = Σ_r U_r·sc_r · rev · Eff
runcost = ρ0 · (1 + ρK·K_eff^1.5) · Σ_r U_r·sc_r     # 高性能ほど提供コスト増
Compute += (income − runcost)                        # dt=1固定
```
- **収入源はCodex採用者のみ**（他社ユーザーは収入にならない＝シェアも大事）。ただし総普及を伸ばすとパイが増え結局効く。
- **ブラウンアウト（死のスパイラル対策・レビュー決着）**: `Compute` が0に達したら `K_eff = min(K, max(2, K × 0.75))` に**一時自動低下**させ、提供範囲の縮退として運用費を通常の25%へ落とす。runcostを抑えつつも高性能モデルの安全・統治負荷は残す。45 PFと全力運転赤字30日分を回復したら `K_eff→K` に戻る。これで操作不能を回復しつつ、ブラウンアウトを絶滅リスク回避に悪用できない。
- 支出 = §6 のアップグレード / feature。

### 5.6 Trust `T`（緩和形＝目標値へ時定数τで漸近・レビュー決着）
微分の積算はドリフト・張り付きを起こすため、**目標値への緩和**に置換:
```
T_target = clamp( 100·( 0.25 + w_div·(1−HHI) + w_s·(S/10) + w_g·(G/10)
                        − monoPenalty − gapPenalty ), 0, 100 )
T += (T_target − T)/τ                                # dt=1固定・時定数τ1個
```
- **monoPenalty** = `relu(S_c − 0.55)²·scale`（自社シェアが過半を大きく超えると急増）＋ `relu(HHI − 0.45)`。
- **gapPenalty** = `w_gap·(CapSafetyGap + CapGovGap)`。
- 時定数τ1個で挙動が読め、係数の微小な不均衡でも0/100に張り付かない。パラメータ圧縮にも寄与。
- **最適は「自社は最強だが独占はしない、安全とガバナンスを性能に追随させる」中庸** — これがF(x)の狙いどおりの均衡。

### 5.7 分岐トリガー（毎tick後判定・共通閾値`θ_gap`＋分岐別スケール）
共通閾値 `θ_gap`（付録B・#10）を超えた分に分岐別スケールを掛ける（パラメータ圧縮）:
- `CapSafetyGap ≥ θ_gap` → 確率 `∝ (gap−θ_gap)·s_acc` で **Safety Incident**（T −大、対象シェア減、flag）。
- `CapSafetyGap ≥ θ_gap` かつ 継続一定日数 → **Misalignment**（終端。**デモモードでは無効化**・§10）。
- `CapGovGap ≥ θ_gap` or `monoPenalty` 高 → **Regulation**（`reg_r` 上昇、成長キャップ）。
- いずれも「gapが開くほど確率・強度が上がる」連続関数。踏みすぎの明確なフィードバック。分岐別スケール `s_acc/s_mis/s_reg` は補助定数（付録B補足）。

## 6. 3アップグレード軸（操作 `u`）
Computeを投資。各購入は**離散アクション**としてエンジンが検証（コスト・上限・副作用を固定表から）。

### 6.1 Model（性能）
- 効果: `K += 1`（上限10）。魅力度・パイMomentum上昇。
- 副作用: `runcost` 上昇、`CapSafetyGap`/`CapGovGap` 拡大 → 事故・規制リスク増。
- コスト: `cost_K(K) = c0·2^K`（指数。踏みすぎを資源面でも抑制）。

### 6.2 Product / Features（機能）★自由テキスト入力の主戦場
- **プリセット機能**（ボタン）: 「モバイルSDK」「エンタープライズSSO」「教育モード」等 → 特定`productFit`ブースト、即時・低リスク。
- **自由入力機能**: プレイヤーが任意テキスト → **ローカルキーワード表が主判定**（数値効果 `productFit`/`brand` を決定）、**GMは演出上乗せ**（`feature_result` のフレーバー・微ナッジのみ、範囲は §7.2/共通constantsでクランプ）。面白い入力ほど大きく報い、ふざけた入力はユーモアニュース＋小効果。
- コスト: 中。**リスク機構は自由入力から独立** — 事故/規制/暴走は §5.7 の `K−S` / `K−G` gap経由でエンジンが専有し、自由入力・GMは触れない（存在しないRisk状態を参照しない）。

### 6.3 Company / Organization（会社構成）＝安全弁
3サブ投資（Computeで購入）:
- **DataCenter** → `Eff` 上昇（Compute収支改善）。
- **Safety Team** → `S += 1`（CapSafetyGap縮小＝事故・暴走の予防）。
- **Policy / Gov Relations** → `G += 1`（CapGovGap縮小＝規制ダメージ軽減）。
プレイヤーは「性能を伸ばす（K）」と「安全弁（S,G,Eff）」の配分ジレンマに常時直面する。

### 6.4 Open Ecosystem / APIパートナー開放（★独占の"正の出口動詞"・レビュー新設）
独占ペナルティに対しプレイヤーの動詞が全て「拡大」しかない矛盾を解消する唯一の"手放す"アクション:
- 効果: 自社世界シェアを一部放出（`sc_r` を一律に少し下げる）→ **Trust大幅+ / パイMomentum+ / HHI低下**。
- 使いどころ: 独占に傾いてTが落ち始めたときの立て直し。**健全帯を狙うと隠れた最適戦略**になり、反独占テーマを"正の動詞"で体現する。
- コスト: Compute小 or 無償・クールダウン制。放出しすぎるとCoverage/収入が痩せるトレードオフ。

### 6.5 トークンリセットボタン（TIBO）
- クールダウン制（例CT=45秒）。押すと一定時間（例8秒）全リージョンの `Momentum_r` とCodex魅力度を大ブースト → 地図一斉発光・数字急騰。
- **Riskは上げない純ご褒美**。ドーパミンの核。通常成長と明確に差をつける。

## 7. GM契約（GM Contract）— LLM/Codexの境界
GMは **Codexサブエージェント**（または当日フォールバックでOpenAI API）。役割は**イベント提案のみ**。状態は書けない。

### 7.1 GMが見るもの（read-only snapshot、10秒周期）
`{ date, A_world, S_c, HHI, T, K, S, G, topRegions[], recentEvents[], playerInbox[] }`（数値は丸め、内部係数は渡さない）。

#### 7.1.1 二経路GMループ
GMの遅延・一時停止に耐えつつ、入力直後の応答性も保つため、役割を二つに分ける。

1. **即時アクション経路:** 機能追加、地域イベント、2029/2035の重大選択を受信した時だけGMを起動。ローカル判定を先に適用し、GMのニュース・解釈は到着後に上乗せするため、応答待ちでゲームを停止しない。
2. **60秒ハートビート経路:** Codexがゲーム状態を定期監視し、環境ニュース、競合・政府・コミュニティの自律反応、未処理入力、GM停止状態を確認する。異常時はスクリプトデッキへフォールバックし、次回ハートビートで復帰する。

ハートビートは即時応答の代替ではなく**世界を動かし続けるウォッチドッグ**。Codex app側で60秒周期を保証できない場合は、ローカルブリッジの60秒タイマーをトリガーとし、同じGMターン契約を呼び出す。

### 7.2 GMが返すもの: **1イベント=1ファイル** `events/evt-<uuid>.json`（atomic書込・レビュー決着）
GMは tmp に書いてから rename（追記・単一jsonへのポーリングは競合するため廃止）。エンジンはディレクトリを走査し、パース失敗ファイルは無視して次回リトライ。**`risk_delta`は削除**（リスクはエンジンが gap 経由で専有・GMは触れない）。**`date`は表示専用**（x8で80日ドリフトするため適用は受信時）。範囲値は**単一定数表 `constants`（付録B）をプロンプト生成とエンジン検証の両方が参照**し、食い違いバグを構造的に封じる。
```json
{
  "id": "evt-<uuid>",
  "date": "<YYYY-MM-DD>",              // 表示専用。効果は受信時に適用
  "type": "news | feature_result | rival | community_event",
  "headline": "<=40 chars, JP",
  "region": "<regionId | 'global'>",
  "effect": {
    "users_delta_pct":   { "min": -30,   "max": 60   },  // 対象地域Uへの一時増減
    "share_delta":       { "min": -0.15, "max": 0.20 },  // sc_r（target=codex）or 競合世界シェア
    "growth_rate_delta": { "min": -0.2,  "max": 0.4  },  // Momentum係数への一時加算
    "trust_delta":       { "min": -8,    "max": 8    },  // T_targetへの一時オフセット
    "target": "codex | rivalAnthro | rivalGoo | rivalQi"
  },
  "flavor": "<=1 sentence",
  "ttl_days": { "min": 1, "max": 30 }
}
```

### 7.3 安全境界（Safety Boundaries）— エンジンが強制
1. **範囲クランプ**: 全数値を `constants`（付録B）のmin/maxにクランプ。範囲外・欠損・型不一致は当該フィールドを0扱い。
2. **不変条件（invariants）**: シェア合計=1を再正規化、`U_r∈[0,P_r]`、`T∈[0,100]`、`Compute≥0` をエンジンが常に保証。GMはこれを壊せない。**リスク機構（gap→事故/規制）はエンジン専有、GM到達不可**。
3. **レート制限**: 1提案サイクルあたり最大3イベント、`users_delta_pct` の合計に上限。連続爆増はトークンリセット専用。
4. **フォールバック**: スキーマ検証失敗 / タイムアウト（>Xs）/ GM不在 → **スクリプト済みイベントデッキ（§8.1）**。ゲームは決して停止しない。
5. **内容ガード**: 実在個人・企業への中傷生成を禁止。競合はぼかし名。**NGワードフィルタを入力・出力の両方**に適用（プロンプトインジェクション＋会場投影事故の両対策）。
6. **決定論の担保**: GMは「範囲内のフレーバー＋微ナッジ」のみ。勝敗を決めるコア均衡はエンジン側の決定論式（§5）が支配。デモは**固定シード＋スクリプトデッキで弧を駆動**、GMはライブだが効果クランプを更に絞った飾り。自由入力は**ローカルキーワード表を主判定**、GMは演出上乗せ（Open Q3決着）。入力は**60字上限**。

## 10. 時間設計
- 内部dt=1日、`speed`∈{1,8}はサブステップ回数（§5）。既定はNormal、プレイヤーがFast Forwardへ切り替える。重要イベントでは現在速度を保ったまま自動停止する。
- **通常プレイ 15分** で 2026→2040 に届くよう、中盤以降の自動加速 or プレイヤー加速を前提にバランス。
- **重要イベント・選択肢発火時は時間を停止**し、原因・Trustへの影響・最上位リスクを表示する。確認後は元の速度で再開する。
- **単一Normalルール**: 自動デモモードは廃止する。初回は4ステップのチュートリアルを表示し、完了まで時間を進めない。チュートリアルはいつでも再表示できる。
- **Momentum**: 放置中の普及成長は基準値の6%に落ち、Computeの運用費と競合進行は続く。機能、地域、投資、Token Reset、Open Ecosystem、重大選択により有限日数のMomentumを再起動する。介入2回未満ではS評価に到達できない。

## 12. 受入基準（Acceptance Criteria・テスト可能）
- **AC1**: 地図が描画され、放置時の `U_r` 成長が基準の6%へ停滞し、機能追加等の介入後に有限日数だけ通常成長へ復帰する。放置中もComputeコストと競合進行は継続する。
- **AC2**: トークンリセット押下で全地域が発光し、押下後8秒間の総ユーザー増分が通常比 ≥3倍。CT中は再押下不可。
- **AC2b**: 内部dt=1固定により、x8＋トークンリセット同時でも全リージョンで `U_r ≤ P_r`（オーバーシュート・振動なし）。
- **AC2c**: Open Ecosystem押下で `S_c`低下・`HHI`低下・`T`上昇が同時に起き、独占ルートからの立て直しが可能。
- **AC3**: Model購入で `K` が上がり、`CapSafetyGap` 増加がゲージに反映。Safety購入で縮小。
- **AC4**: 自由入力「モバイル対応」等でモバイル比率高リージョンの `sc_r` が有意に上昇（低比率地域より大きく）。
- **AC5**: GM/API不在でもスクリプトデッキで速報が流れ続け、ゲームが停止しない（GMプロセスをkillして確認）。
- **AC6**: GMがレンジ外の巨大effectを返してもエンジンがクランプし、シェア合計=1・T∈[0,100]・Compute≥0 の不変条件が破れない。**途中切断された不完全JSONファイルを読んでも例外を出さず無視して次回リトライ**（atomic書込テスト）。
- **AC7**: 独占ルート（自社シェアを過剰に上げる）で `monoPenalty`→`T`低下→Pyrrhic/Regulation分岐に入り、**Sランクにならない**。
- **AC8**: バランス調整で健全帯（自社強・HHI健全・安全追随）が最高スコアになる（少なくとも1本のプレイスルーで Sランク到達）。
- **AC9**: 初回表示で4ステップのチュートリアルが開き、完了まで日付は進まない。最終ステップの開始操作後のみ単一Normalルールで進行し、HOW TO PLAYから再表示できる。
- **AC10**: 決定論: 同一シード・同一入力列で2回実行して最終状態が一致（GM無効時）。
- **AC11**: Compute=0でブラウンアウトが発動し `K_eff` 低下→runcost減→Compute回復→`K_eff→K` 復帰。詰みが発生しない。
- **AC12**: Codex appの左ペインGMと右ペイン内ブラウザが協調し、右で送った教育モード提案に対して、ローカル効果は即時反映され、GMの便益と統治課題を含むイベントが後から安全に上乗せされる。
- **AC13**: 60秒ハートビートが状態を監視し、プレイヤー操作がなくても世界イベントが進む。GMを一時停止してもゲームはスクリプトデッキで継続し、次回利用可能時に復帰する。
- **AC14**: 2029年と2035年の選択が最終ルートを実際に変え、Plan A条件を満たさないプレイが自動的にBeneficial Abundanceへ到達しない。
- **AC15**: 主要ストーリーイベントが `AI 2027` / `AI 2040` / `Your Timeline` / `Live GM` のいずれかに明確に分類され、原典由来と生成上の推論をプレイヤーが区別できる。
- **AC16**: エンディングでReference ScenarioとYour Timelineの分岐点を振り返れ、少なくとも3つの選択について「何が起き、なぜ重要だったか」を確認できる。
