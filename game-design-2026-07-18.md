# ゲームデザイン: 状態モデルとF(x) — Codex Pandemic
2026-07-17 Lead作成。Kaiの2つの問い（①F(x)/操作軸、②独占ゴールの違和感）への回答＋数式設計。

## 結論を先に: 独占の違和感は「勝利条件の再定義」で解ける
Kaiの直感は正しい。「全世界をOpenAIで塗りつぶす＝独占」は (a) OpenAIの前で流すには筋が悪く、(b) ゲームとしても「無双で飽きる」。
**解決策: 勝利条件を『市場シェア独占』から『信頼を保ったまま人類全体にAIを行き渡らせる』に変える。**
- スコアは「普及率（Adoption）」。地図がブワッと光る快感はそのまま残る（＝人類にAIが届く演出であって、他社を潰す演出ではない）。
- ライバル（Anthropic系・Google系・中国モデル系）は**倒す敵ではなく、生かすほど有利**にする。健全な競争がある＝**Trust（社会的信頼）ゲージが上がる**。独占に傾くと逆に規制・反発で失速。
- これで「競争が市場に良い」というKaiの現実感覚が、そのままゲームの勝ち筋になる。エンディングで一言ウィンク: *「君は世界を"所有"して勝ったんじゃない。AIを全員の手に届けて勝った。ライバル？ 彼らが君を強くした。」*
- ※もし当日「純粋な世界征服の快感」に振りたくなったら、皮肉・セルフパロディとして振り切る手もある（config1つで切替可能に作る）。ただし推奨は上記の"beneficial adoption + trust"版。OpenAI審査・観客投票の場に最も合う。

## これがそのままF(x)のドラマになる: 「速さ vs 安全」のトレードオフ
Kaiのメモ「性能を上げすぎても規制で停められる」＝ゲームの緊張の源。単なる右肩上がりでなく、**踏むほど事故る**構造にする。AI2027の暴走シナリオ＝この緊張の最大化。

## 状態変数（最小構成）
- **地域ごと**: 人口 `P_r`（固定）、採用者数 `A_r`（0〜P_r）、地域成長率 `g_r`
- **グローバル**: `Compute`（唯一の資源）、`Capability K`（モデル性能レベル）、`Trust T`（0〜100）、ゲーム内時刻 `t`
- **資源は Compute（電力/GPU）1本に統一を推奨**（Kaiの「資金か電力か」への回答）。理由: AIの現実の制約そのもので世界観に合う／「上限がある」緊張を自然に生む。裏側の抽象が"お金"だが、プレイヤーに見せるのは Compute 1種に絞る（3時間ビルドでは資源は1つが鉄則）。

## F(x): ティック更新（1秒 = ゲーム内1日）
```
# 1) 普及はロジスティック成長（Plague Incの「静→爆発→飽和」の気持ちよさの正体）
A_r += g_r * A_r * (1 - A_r / P_r) * K_mult * TrustFactor
   K_mult    = 1 + 0.15 * K            # モデル性能が成長を底上げ
   TrustFactor = 0.5 + T/200           # 信頼が低いと伸びが鈍る(0.5〜1.0)
   # 地域は「導入」されるまで A_r=0（クリック/イベント/自然波及でシード）

# 2) Compute 収支
Compute += Σ(A_r) * rev_per_user * OrgEfficiency  -  running_cost(K)
   # 採用者が増えるほど収入増。ただしKが高いほど維持費増（踏みすぎ抑制）

# 3) Trust / Risk（速さ vs 安全）
Risk  += ΔK * risk_per_level  +  reckless_feature_risk  -  SafetyInvest  -  RivalHealthBonus
T = clamp(100 - Risk, 0, 100)
   RivalHealthBonus: 生きているライバル社数に比例（＝競争が信頼を守る＝反独占の実装）
   Risk 高 → 規制イベント発火確率↑、暴走インシデント確率↑
```
ロジスティック `A(1−A/P)` が Plague Inc 感の本体。ここだけは必ず入れる。

## 3つのアップグレード軸（Kaiのメモの実装）
プレイヤーが Compute を投じて更新する3ツリー:
1. **Model（性能 K）**: 成長天井とアンロックを上げる。**ただし Risk が上がる**。上げすぎ＋安全投資不足＝暴走ルートへ。
2. **Product/Features（機能）**: **例のテキスト自由入力の欄がここ**。「Codexがスマホ対応」等を入力→GMが効果判定→特定地域/セグメントの `g_r` にブースト。創造性が報われる主役メカニクス。ふざけた入力は小効果＋ユーモアニュース。
3. **Company/Org（会社構成）**: データセンター（Compute収入↑）、Safetyチーム（Risk↓）、Policy/渉外（規制ダメージ↓）、DevRel（波及速度↑）。**安全と持続性の軸**＝暴走を避けるための投資先。

## ライバル（Cua = 競合）
Anthropic系・Google系・中国モデル系の3社。各地域でシェアを持ち、成長し、時々あなたの地域を"contest"（`g_r` を一時減衰）→ プレイヤーはイベント/機能で応戦。
**倒す必要はない。生存＝Trust加点（反独占ウィンク）。** 応戦のリアクション性だけで十分な緊張が出る（Plague Incの「治療薬開発」枠の役割）。

## トークンリセットボタン（Tibo）＝ドーパミンの核
クールダウン制の必殺技。押すと一定時間 全地域 `g_r` が急騰＝地図が一斉発光・数字が跳ねる。通常成長とははっきり差をつけ、「押せる」快感を突出させる。Risk は上げない（純粋なご褒美）。

## エンディング分岐（AI2027/2040 の弧・デモの山場）
- **勝利（Abundance / Safe AGI）**: 2040到達時に 世界普及率 ≥ 閾値（例60-80%）かつ T ≥ 下限。「信頼を保ったまま人類全体に届いた」リザルト（総採用者・開催イベント数・追加機能一覧を表示）。
- **暗黒（Misalignment）**: K が高い×Safety不足で暴走インシデント連鎖 → AI2027ダーク版。Kaiの言う「超知能の暴走」ルートはここ。
- **失速（Regulatory Freeze）**: 独占に傾き Trust 崩壊 → 規制カスケードで採用者急落。
3分岐あるとデモで「このゲーム、負けもあるんです」が言えて深みが出る。

## 3時間での実装優先度（前出フォールバック階段と対応）
- **必須(L0-L1)**: 地域 `A_r` のロジスティック成長＋Compute＋地図発光＋カウンター＋トークンリセットボタン。**この時点でF(x)の気持ちよさは完成**。
- **L2**: 3アップグレードツリー（Model/Features/Org）＋ Trust/Risk ゲージ＋機能テキスト入力のLLM判定。
- **L3**: GM常駐（リアルタイムイベント）＋ライバル社の contest。
- **L4**: 3エンディング分岐＋速度調整＋AI2040タイムスキップ演出。
数値（0.15, risk_per_level 等）は当日 30秒プレイして体感で詰める。デモが15分で 2040 に届くよう時間速度で調整。
